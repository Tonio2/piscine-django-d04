Django

  1. Moteur de templates Django :
    • Le moteur de templates Django utilise la notation {% verbatim %}{{ variable }}{% endverbatim %} pour insérer des variables dans un modèle HTML. Ces variables sont généralement fournies dans un contexte par la vue.
    • Les modèles peuvent également contenir des balises et des filtres spéciaux qui permettent de manipuler et de formater les données affichées.
    • Les templates sont enregistrés dans des fichiers séparés avec l'extension .html.
  2. Blocs :
    • Les blocs sont utilisés pour définir des zones de contenu réutilisables dans les templates. Ils sont définis avec la syntaxe {% block nom_du_bloc %} et {% endblock %}.
    • Les modèles qui étendent d'autres modèles peuvent remplacer ou étendre ces blocs en définissant à nouveau le contenu du bloc dans le modèle enfant.
  3. Boucles for ... in :
    • Les boucles for ... in permettent de parcourir des listes ou des ensembles de données dans un modèle HTML.
    • La syntaxe est la suivante : 
                {% verbatim %}{% for element in liste %}
            <!-- Code HTML utilisant l'élément -->
          {% endfor %}{% endverbatim %}
  4. Structures de contrôle if :
    • Les structures de contrôle if permettent de conditionner l'affichage de contenu dans un modèle HTML en fonction d'une condition.
    • La syntaxe est similaire à celle du langage Python : 
                {% verbatim %}{% if condition %}
            <!-- Code HTML à afficher si la condition est vraie -->
          {% else %}
            <!-- Code HTML à afficher si la condition est fausse -->
          {% endif %}{% endverbatim %}
  5. Affichage de variables passées par contexte :
    • Les variables passées par contexte depuis la vue sont accessibles dans les templates avec la notation {% verbatim %}{{ variable }}{% endverbatim %}.
    • Par exemple, si une vue passe un dictionnaire de contexte contenant une variable nom à un template, vous pouvez l'afficher dans le modèle ainsi : {% verbatim %}{{ nom }}{% endverbatim %}.
